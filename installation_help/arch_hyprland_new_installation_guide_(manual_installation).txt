Syntax highlighting:
Nano ships with predefined syntax highlighting rules, defined in /usr/share/nano/*.nanorc and /usr/share/nano/extra/*.nanorc. To enable them, add the following line to your ~/.config/nano/nanorc or to /etc/nanorc:
include "/usr/share/nano/*.nanorc"
include "/usr/share/nano/extra/*.nanorc"
For syntax highlighting enhancements which replace and expand the defaults, install nano-syntax-highlighting or nano-syntax-highlighting-gitAUR and, additionally to the above setting, also add:
include "/usr/share/nano-syntax-highlighting/*.nanorc"

Automatically Mounting Using udiskie:
Starting method: manual (’exec-once')
USB mass storage devices, like thumb drives, mobile phones, digital cameras, etc. are not mounted automatically to the file system.
Typically, they have to be manually mounted, often using root and umount to do so.
Many popular DEs automatically handle this by using udisks2 wrappers.
udiskie is a udisks2 front-end that allows to manage removable media such as CDs or flash drives from userspace.
Install udiskie via your package manager, or build manually
Head over to your hyprland.conf and add the following line:
exec-once = udiskie

That file ensures that getty logs you in automatically on tty1.

====== ARCH LINUX AUTO LOGIN (ANOTHER WAY WHEN SDDM NOT INSTALLED) ======

# Open a terminal and create a systemd service for auto-login:
$ sudo nano /etc/systemd/system/auto-login@.service

Paste the following content into the file:

[Unit]
Description=Automatic login for user %i
After=systemd-user-sessions.service

[Service]
ExecStart=-/usr/bin/login -f %i
StandardInput=tty
TTYPath=/dev/tty1
Restart=always

[Install]
WantedBy=multi-user.target

Save and exit

# Enable the auto-login service for your user (replace username with your actual username):
$ sudo systemctl enable auto-login@andres.service

Reboot to test

====== DISABLE WATCHDOG SERVICES ======

# Create or edit the system.conf file:
$ sudo nano /etc/systemd/system.conf

Change these values:

[Manager]
RuntimeWatchdogSec=0
ShutdownWatchdogSec=0
RebootWatchdogSec=0
RuntimeWatchdogPreSec=0  # Optional, but ensures no pre-shutdown watchdog triggers

# Reload the systemd configuration:
$ sudo systemctl daemon-reexec

# After rebooting, confirm that watchdog functionality is disabled:
$ cat /proc/sys/kernel/watchdog

Output should be 0.

====== ENABLING PERFORMANCE MODE ======

# Install cpupower:
$ sudo pacman -S cpupower

# Then enable and start the service:
$ sudo systemctl enable --now cpupower.service

# Then edit /etc/default/cpupower:
$ sudo nano /etc/default/cpupower

Change:
governor='performance'

# Save and reboot. Save and exit. Then reboot and check if it's applied with:
$ cat /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor

If it prints "performance", it's working.
